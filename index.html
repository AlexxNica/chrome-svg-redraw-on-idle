Loading can take 5 secs...

If you have a ton of svgs on a page, they all get rendered and composited on every frame anytime you touch the DOM, even if that DOM modification doesn't effect the svgs at all. Here, the only javascript running after initialization is to update the fps counter, and yet CPU usage and chrome redraw suffer continuously. This affects scrolling performance and other script execution speed.

Setting a hack of a 3d transform on the SVG elements clears up the issue and restores 60fps and drastically reduces CPU usage.

One would expect the initial rendering to be slow, of course, but the subsequent, continuous rendering overhead seems accidental.
